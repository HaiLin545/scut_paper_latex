# 第四章方法整体框架图绘制 Prompt

## 一、图片基本信息

- **图片标题**：基于人体几何先验的服装参数生成方法整体框架
- **对应论文章节**：第四章 §4.4 方法整体框架（`\label{fig:ch4-framework}`）
- **输出格式**：矢量 PDF，宽度适配 LaTeX `\linewidth`（约 16cm），高度约 8–10cm
- **绘制工具建议**：Adobe Illustrator / Figma / TikZ / draw.io 均可；推荐 Illustrator 或 Figma 以获得最佳的学术美感

---

## 二、整体布局

采用**从左到右的水平流水线布局**（pipeline-style），共分为 **5 个主阶段**，用带箭头的连线串联。各阶段之间保持等距。整个图按功能可划分为三个水平带（lane）：

| 层级 | 内容 | 位置 |
|------|------|------|
| 上方注释带 | 各模块的简要功能标签 | 各模块正上方 |
| 主流水线带 | 核心模块（方框/圆角矩形）+ 中间数据产物 | 图的中部，占主要空间 |
| 下方辅助带 | 提示词构建三层文档的展开细节 | 第三阶段下方局部展开 |

整体从左到右的阶段划分：

```
[输入图像] → [阶段1: 人体参数估计与身体测量提取] → [阶段2: 紧凑格式设计] → [阶段3: 规范化提示构建 + 多模态联合推理] → [阶段4: 参数转换与三维服装构建] → [输出: 3D服装]
```

---

## 三、各阶段详细内容

### 阶段 0：输入（最左侧）

- **可视化元素**：一张示意性的服装穿着图片（人物穿衣全身照的缩略图轮廓）
- **标注文字**：「输入图像 $\mathbf{I}$」
- **样式**：带细边框的图片缩略图，左侧无箭头输入

---

### 阶段 1：人体参数估计与身体测量提取

此阶段包含两个子步骤，**上下排列或左右紧凑排列**：

#### 子步骤 1a：SMPL 参数估计
- **方框内容**：
  - 图标/示意：一个简化的 SMPL 人体网格轮廓（线框风格的人体 mesh）
  - 文字标注：「SMPL 估计（ROMP）」
- **输入箭头**：从输入图像引入
- **输出数据标注**：在箭头上或旁边标注「$\boldsymbol{\beta}, \boldsymbol{\theta}$」

#### 子步骤 1b：身体测量提取
- **方框内容**：
  - 图标/示意：人体轮廓上标注若干测量线段（如臂长、腰围的虚线/尺寸标注线）
  - 文字标注：「GarmentMeasurements（SMPL 适配）」
- **输入箭头**：从 SMPL 网格引入
- **输出数据标注**：在方框右侧或输出箭头旁列出关键测量项的小标签：
  ```
  身体测量 b:
  · arm_length
  · bust / waist
  · leg_length
  · ...（共28项）
  ```

**★ 创新点高亮**：在子步骤 1b 方框底部或旁边加一个**星标/高亮标注条**，写明：
> 「贡献④：SMPL 跨拓扑量体适配」

---

### 阶段 2：面向 LLM 的设计参数紧凑化

- **可视化方式**：用一个**对比示意**来突出紧凑化的效果：
  - 左侧小框：原生格式（标注「~886 行 YAML」），用密集的灰色横线表示冗长文本
  - 一个向右的压缩箭头（带**信息分流**标签）
  - 右侧小框：紧凑格式（标注「≤40 行 YAML」），用稀疏的几行彩色文字表示
  - 从原生格式向下分出一条虚线箭头，指向阶段 3 的「参数规格文档」，标注「元数据迁移至提示文档」

- **方框标题**：「紧凑参数格式」

- **紧凑格式框中可展示的示意文本**（缩小字体，仅作视觉示意，不需完全可读）：
  ```yaml
  meta:
    upper: Shirt
    bottom: null
  shirt:
    length: 1.2
    width: 1.05
  sleeve:
    length: 0.3
  ```

**★ 创新点高亮**：
> 「贡献②：信息分流 → 输出规模压缩 90%+」

---

### 阶段 3：规范化提示构建 + 多模态联合推理（核心区域，占最大宽度）

此阶段是全图的**视觉重心**，建议用**稍大的区域**和**更丰富的细节**呈现。分为上下两部分：

#### 上半部分：三层提示构建

用**三个平行的文档卡片**（竖向排列或扇形展开）表示系统提示的三层结构：

| 文档卡片 | 标题 | 简要内容示意 | 颜色建议 |
|----------|------|-------------|---------|
| 卡片 1 | 主提示模板 | 「角色定义 · 五步工作流 · 规则约束」 | 蓝色系 |
| 卡片 2 | 身体测量文档 | 「测量语义 · 映射关系 · 数值示例」+ 从阶段1的 b 引入的箭头 | 绿色系 |
| 卡片 3 | 参数规格文档 | 「参数名 · 类型 · 范围 · 激活规则 · 示例」+ 从阶段2引入的虚线箭头 | 橙色系 |

三张卡片通过大括号或汇聚箭头合并为一个「系统提示（System Prompt）」。

**★ 创新点高亮**：
> 「贡献③：三层结构化提示体系」

#### 下半部分：用户提示 + GPT-4o 推理

- **用户提示框**：包含两个输入——
  - 「身体测量 YAML」（从阶段 1 的输出引入，绿色边框）
  - 「服装图像」（从最左侧输入图像引入一条长虚线，或标注"同一张输入图像"）

- **GPT-4o 推理核心框**（整个阶段的中心元素）：
  - 用一个**大号圆角矩形**，内部写「GPT-4o \n 多模态联合推理」
  - 可加一个 GPT/AI 的小图标
  - 框内或框旁用小字标注推理链的四步：「视觉分析 → 类型映射 → 比例估计 → 参数输出」
  - 左侧接收系统提示 + 用户提示两条输入
  - 右侧输出一个**紧凑 YAML 格式的设计参数 d**

**★ 核心创新高亮**（本章最关键的创新点，建议用最醒目的方式标注）：
> 「贡献①：人体几何先验注入 → 免训练单次推理参数生成」

在从「身体测量 b」到「GPT-4o」的箭头上，用**加粗/高亮色/虚线框**重点标注：
> 「几何先验注入」

---

### 阶段 4：参数转换与三维服装构建

此阶段包含三个子步骤，从左到右依次排列：

#### 子步骤 4a：格式还原
- **方框内容**：「紧凑格式 → 完整格式 \n（类型安全转换）」
- **输入**：GPT-4o 输出的紧凑 YAML
- **输出**：`design.yaml`（完整格式）

#### 子步骤 4b：二维样板生成
- **方框内容**：「GarmentCode 制版引擎」
- **图标/示意**：二维缝纫样板的简化轮廓（几个不规则多边形拼排，如上装前片/后片/袖片的简笔画）
- **输入**：`design.yaml` + `smpl.yaml`（后者从阶段 1 引入）

#### 子步骤 4c：三维服装构建
- **方框内容**：「物理仿真」
- **图标/示意**：穿着于人体上的三维服装网格（简笔画风格）
- **输出**：最终的三维服装网格 $\mathcal{G}_{\text{garment}}$

---

### 阶段 5：输出（最右侧）

- **可视化元素**：两个并排的输出结果示意——
  - 左：二维缝纫样板（平面裁片展开图）
  - 右：三维穿着效果（人体 + 服装网格）
- **标注文字**：「二维缝纫样板」「三维服装网格」

---

## 四、连线与数据流

所有连线使用**带箭头的实线**（主数据流）或**虚线**（辅助数据流/元数据迁移）。关键数据流标注如下：

| 起点 | 终点 | 标注 | 线型 |
|------|------|------|------|
| 输入图像 | SMPL 估计 | — | 实线 |
| 输入图像 | GPT-4o（用户提示中的图像） | 「图像」 | 虚线（长距离跨越） |
| SMPL 估计 | 身体测量提取 | 「SMPL 网格 $\mathcal{M}$」 | 实线 |
| 身体测量提取 | 身体测量文档（提示卡片 2） | 「$\mathbf{b}$」 | 实线 |
| 身体测量提取 | 用户提示 | 「$\mathbf{b}$ (YAML)」 | 实线 |
| 身体测量提取 | 三维服装构建（4b） | 「$\mathbf{b}$ (smpl.yaml)」 | 虚线（长距离跨越） |
| 紧凑格式设计 | 参数规格文档（提示卡片 3） | 「元数据」 | 虚线 |
| GPT-4o | 格式还原 | 「紧凑 YAML $\mathbf{d}$」 | 实线 |
| 格式还原 | GarmentCode | 「design.yaml」 | 实线 |
| GarmentCode | 物理仿真 | 「2D 样板」 | 实线 |
| 物理仿真 | 输出 | 「3D 网格 $\mathcal{G}$」 | 实线 |

## 五、色彩风格

采用**学术论文常用的低饱和度柔和配色**，整体风格干净、专业、信息密度高但不杂乱。

### 配色方案

| 元素 | 颜色 | 色值参考 (HEX) | 用途 |
|------|------|--------------|------|
| 阶段 1 模块（人体相关） | 柔和蓝绿 | `#E8F4F8` 填充 / `#5B9BD5` 边框 | SMPL 估计、身体测量 |
| 阶段 2 模块（格式设计） | 柔和橙色 | `#FFF2E0` 填充 / `#ED7D31` 边框 | 紧凑格式 |

| 提示文档卡片 1（主提示） | 浅蓝 | `#DBEAFE` 填充 / `#3B82F6` 边框 | |

| 背景 | 纯白 | `#FFFFFF` | |
| 箭头与文字 | 深灰 | `#333333` | |
| 虚线（辅助数据流） | 中灰 | `#999999` | |

### 字体与排版

- **模块标题**：无衬线字体（如 Helvetica Neue / Arial / Source Han Sans），加粗，9–10pt
- **模块内说明文字**：同字体，常规，7–8pt
- **数据流标注**：斜体或常规，7pt
- **数学符号**：使用 LaTeX 风格的斜体（如 $\mathbf{b}$, $\mathbf{d}$, $\mathcal{M}$）
- **创新点标注**：加粗 + 红色/品红，8pt

## 六、创新点标注策略

在图中通过**四个带编号的高亮标注**突出本章的四项贡献，建议使用统一样式的标签：

| 标注编号 | 内容 | 位置 |
|---------|------|------|

## 七、设计准则与注意事项

1. **信息层次分明**：主流水线的模块框和箭头在视觉上最突出；辅助数据流（虚线）退居次要；创新点标注作为第三层级的注意力引导
2. **避免视觉拥挤**：模块之间保持足够间距（建议至少 1cm）；长距离跨越的虚线使用曲线避让，不要穿越其他模块
3. **对齐与对称**：所有模块框的高度尽量统一；箭头保持水平或垂直（45° 也可接受）；避免倾斜的文字标注
4. **图中文字语言**：与论文正文一致，使用**中文**标注模块名和说明，但专有术语（SMPL、YAML、GPT-4o、GarmentCode 等）保留英文
5. **可独立理解**：框架图应能让读者仅看图即理解方法全流程，无需借助正文
6. **印刷兼容**：配色方案应在黑白打印时仍能通过亮度差异区分不同模块（避免仅靠色相区分）

## 九、补充说明

- 如果图片高度受限，可将阶段 3 中的三张提示文档卡片改为水平排列而非竖向堆叠
